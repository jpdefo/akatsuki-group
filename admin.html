Publish snapshot

GitHub Pages is read-only. Importing JSON here updates your browser locally; publishing is what makes the public site change.

One-command publish flow: publish-snapshot.cmd Save current overrides first if you want HLTB, 10%, month, extra giveaway, or pause edits to become part of the next public snapshot. First-time setup on a fresh clone: run setup-publish.cmd once. It starts the local server, opens the bookmarklet helper and SteamGifts, waits for a fresh collector sync, refreshes Steam data, and publishes automatically.
